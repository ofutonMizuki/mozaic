# mozaic — Vision

> TypeScript 風の構文を出発点に、**CPU と GPU を同格に扱う**システムプログラミング言語。
> 動作速度が最優先。書き手の快適さより、出力されるコードの速さを取る。

*※ 本書のコードは構文イメージです。文法は M0 で確定させます。*

## 一行で

**「TypeScript を出発点に、Zig/Rust の中身」** — 親しみやすい構文を基調に、意味論はシステム言語。
1 つのソースが CPU でも GPU でも同格に動き、C++ / WASM / JS へ変換される。

TypeScript は **親しみやすさの出発点であって制約ではない**。速度・厳密な型・所有権・CPU/GPU 同格に
資するなら、Rust / Zig など他言語の構文を取り入れ、TS と違ってよい(例:`&T` / `&mut T` 参照、
固定幅型、`struct` / `enum` / `match`、実変換としての `as`)。

## 哲学 / なぜ作るのか

- 並列計算機の本質は「複数の演算器が同じメモリを共有して働く」こと。とりわけ Apple Silicon の
  ユニファイドメモリでは、CPU と GPU は物理的に対等な隣人になった。既存言語の多くは依然 GPU を
  「遠い別世界」として特別扱いする。mozaic は **CPU と GPU を最初から対等な実行デバイス**として
  言語に組み込む。
- 速いコードは、隠れたコスト(GC・隠れた確保・動的ディスパッチ)が無いことから生まれる。
  mozaic はそれらを言語から排し、コストを常に目に見える形にする。

## 設計の柱

1. **同格 (Peer devices)** — CPU と GPU は同じインターフェースを持つ `Device`。カーネルは一語の差し替えで両者を行き来する。
2. **所有権と借用** — メモリ安全を GC なしで実現。さらに **借用が CPU/GPU 間の同期そのものを表す**(下のキーストーン参照)。
3. **厳密な静的型** — 固定幅数値、暗黙変換なし、null 無し。
4. **コンパイル時計算 (comptime)** — テーブル生成・特殊化・ゼロコスト総称型。
5. **極小ランタイム** — stdin/stdout とメモリとスレッドだけ。OS 依存を最小化し移植性を稼ぐ。
6. **線形メモリ** — フラットなメモリモデルが C++ / WASM / JS への一様な変換を可能にする。
7. **並行性を初めから (Concurrency-first)** — マルチスレッドは一級市民。同じ借用規則がスレッド間のデータ競合もコンパイル時に封じ、タスク並列(スレッド)とデータ並列(カーネル)を1つのモデルに統一する。

## 非目標 (Non-Goals)

- 書き手の手軽さ・初心者向けスクリプト体験は **狙わない**。速度と明示性を優先する。
- ガベージコレクションは **持たない**。
- JS 出力は速度ターゲットではない(デバッグ・移植用)。ブラウザ性能の本命は WASM。
- 完全な動的型・リフレクション・実行時メタプログラミングは持たない。

## キーストーン:借用 = デバイス同期

mozaic の核心。所有権・借用とユニファイドメモリを掛け合わせると、**CPU/GPU の競合がコンパイル時に消える**。

- `Buffer` は「どこにあるか」ではなく「**いま誰がアクセス権(借用)を持つか**」で管理される。
- カーネル起動はバッファを借用し、その借用を **完了ハンドル `Job` が保持**する。
- ジョブ完了前に同じバッファを CPU から触るコードは **コンパイルエラー**。
- 借用が CPU へ戻る一点が、唯一の同期境界。

同じソースで、コード生成だけが変わる:

| ハードウェア | 借用の返却 |
|---|---|
| ユニファイドメモリ (Apple Silicon) | フェンスのみ・**コピーゼロ** |
| ディスクリート GPU | コンパイラがコピーバック (D→H) を挿入 |

```typescript
let input  = Buffer.shared<f32>(N);
let output = Buffer.shared<f32>(N);

fill(&mut input, source);                         // CPU が可変借用 → 返却

const job = dev.launch(scale, { grid: N }, 2.0, &input, &mut output);
// use(output[0]);   // ← コンパイルエラー: output is mutably borrowed by a pending GPU job
job.await();                                      // UMA ならフェンスのみ(コピー無し)
use(output[0]);                                   // OK
```

## 並行性:1つのモデル、2つの形態

マルチスレッドは初めから一級。**所有権・借用がスレッド間データ競合もコンパイル時に防ぐ**
(キーストーンと同じ仕組み)。並行な仕事はすべて「完了まで借用を保持し、合流点が唯一の同期境界」:

| 形態 | 構文 | 合流(借用返却) | 走る場所 |
|---|---|---|---|
| データ並列 | `dev.launch(k, …)` → `Job` | `job.await()` | CPU プール / GPU |
| タスク並列 | `scope { spawn f(…); }` → `Task` | `task.join()` / scope 終端 | CPU スレッド |

`Device.cpu` への launch は CPU ワーカープールで実行される ── つまり **データ並列マルチスレッドは
「同格」から自動的に手に入る**。Apple Silicon の UMA では、CPU スレッドと GPU が共有メモリ上の
データを借用チェック付きで受け渡せる。

## 実行モデル:二層

| 層 | 性格 | 走る場所 |
|---|---|---|
| **ホスト層** | 言語フル機能。再帰・スレッド・stdio。計算を「指揮」する | CPU |
| **カーネル層** | 確保なし・構造化制御フローの制限サブセット。**同一コードが CPU/GPU 両方で動く** | CPU **も** GPU **も** |

カーネルサブセットは実質 `Metal(MSL) ∩ CUDA ∩ WebGPU(WGSL)` の共通部分。
再帰・動的確保・GC・クロージャは禁止。これが移植性の背骨になる。

## メモリモデル

- **線形メモリ**。C++→生ポインタ / WASM→線形メモリ / JS→ArrayBuffer に一様に落ちる。
- **単一所有 + ムーブ**。所有者がスコープを抜ければ解放 (RAII)。`defer` で明示後始末。
- **借用**:`&T`(共有)/ `&mut T`(排他)。参照は所有者より長生きできない。
- **デバイス常駐**:`Buffer.shared<T>`(CPU/GPU コヒーレント、UMA でゼロコピー)/
  `Buffer.device<T>`(GPU 専有、最速)。常駐は性能ヒントで、借用モデルはどちらも同一。

## 型システム(厳密)

- スカラ:`i8 i16 i32 i64 i128` / `u8 u16 u32 u64 u128` / `f16 f32 f64` / `bool`
- ベクタ(SIMD 兼 GPU レーン):`f32x4`, `i16x8`, `u64x2` …
- 集約:`struct`(値型)/ 固定長配列 `[T; N]` / スライス `[]T` / `Buffer<T>`
- 直和:ペイロード付き `enum` + 網羅 `match`
- オプショナル `T?`(null 無し)
- **暗黙の型変換は一切認めない(いかなる場合も)**。型が違えば必ず明示変換が要る。
  - 数値の拡大も禁止:`let b: i64 = i32val` は不可、`i32val as i64` と書く。
  - 条件は `bool` 限定:`if (n)` は不可、`if (n != 0)`。truthy/falsy は無い。
  - `enum`↔整数、`u8`↔文字 なども明示メソッド/キャスト経由。
  - 「変換ではないもの」は対象外:数値リテラルは文脈の型として生まれる(`let x: f32 = 1` は可)。参照のアクセス解決(自動デリファレンス)も値変換ではない。

```typescript
struct Bitset128 {       // 128 ビットの集合
  lo: u64;
  hi: u64;
  fn test(&self, i: u32): bool {
    return i < 64 ? ((self.lo >> i) & 1) != 0
                  : ((self.hi >> (i - 64)) & 1) != 0;
  }
}
```

## コンパイル時計算 (comptime)

ルックアップ表・定数・特殊化を **コンパイル時に生成**。重いテンプレートを使わずゼロコスト総称型を実現する。

```typescript
const SIN: [f32; 256] = comptime computeSinTable();  // 実行時コスト 0
```

## バックエンド

| ターゲット | ホスト層 | カーネル層 | 状態 |
|---|---|---|---|
| **C++ / Metal**(最初) | C++ + `metal-cpp` | MSL、`MTLStorageModeShared` | 着手 |
| WASM | wasm + threads/SIMD | WebGPU (WGSL) | 後 |
| JS | JS + ArrayBuffer | WebGPU (WGSL) | 後(デバッグ/移植) |
| (将来) CUDA / SYCL | C++ | PTX / SPIR-V | 後 |

カーネル層は 1 つ。降ろし先を足すだけで対応環境が広がる。

## ロードマップ

- **M0 — ホストのみ縦割り**:mozaic → C++。固定幅型・最小の所有権・stdin/stdout で
  「行エコー(`quit` で終了)」を通す。ツールチェーンと線形メモリを固める。
- **M1 — カーネル投入**:`saxpy` 級カーネルを CPU と Metal(MSL)の両方へ降ろし、
  **UMA ゼロコピー借用を実証**。同格の最初の証明。
- **M2 — 言語の肉付け**:`struct` / `enum` / `match`、comptime、ベクタ型。
  標準入力を処理する実用的な小プログラムを書けるところまで。
- **M3 — 借用チェッカ強化** と **WASM/WebGPU バックエンド**。
- **→ セルフホスト**(mozaic を mozaic 自身で書く)を目指す。
